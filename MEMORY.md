## 架构决策

### 消息总线降级策略 (2026-04-08)
**决策**: Kafka 不可用时自动降级到 Redis Pub/Sub

**原因**:
- Docker Hub 网络访问受限
- Kafka 镜像拉取失败
- 需要快速启动系统

**实现**:
- `core/kafka/producer.py` 自动检测端口连通性
- Redis 作为 fallback 后端
- 内存模式作为最后降级

**后续**:
- 网络恢复后可无缝切换回 Kafka
- 性能测试待进行
