# 2026-04-09 - AM-HK v2.0 Kafka Migration

## 今日关键进展

### 1. Agent1 采集问题修复
- **问题**: `_publish_us_data` 方法未定义导致美股数据无法发布
- **修复**: 分离 `_publish_hk_data` 和 `_publish_us_data` 为两个独立方法
- **文件**: `agents/agent1_harvester/main.py`

### 2. RSA 私钥格式修复
- **问题**: `.env` 中 RSA 私钥多行格式导致 `python-dotenv` 解析失败
- **修复**: 私钥转为 base64 编码，代码中自动解码
- **文件**: `.env`, `core/connectors/tiger.py`

### 3. Redis → Kafka 迁移
- **决策**: 从 Redis Pub/Sub 迁移到 Kafka 以提高可靠性
- **原因**: 
  - Kafka 磁盘持久化 vs Redis 内存存储
  - Kafka 原生分布式 vs Redis 需 Sentinel
  - 更好的数据回放能力
- **实施**:
  - 备份原项目: `am-hk-redis-backup-20260409-085046`
  - 拉取 v2.0 Kafka 版本
  - 本地安装 Kafka (端口 9112) + Zookeeper (端口 2181)

### 4. Dashboard 修复
- **问题**: Dashboard 使用 Redis Pub/Sub，与 Kafka 不兼容
- **修复**: 改为使用 `aiokafka` AIOKafkaConsumer 异步消费
- **文件**: `dashboard/main.py`
