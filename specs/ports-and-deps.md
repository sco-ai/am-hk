# 项目规范 - 端口分配

## 端口偏移规则
**基础偏移：+20**（基于默认端口）

| 服务 | 默认端口 | 项目分配端口 | 用途 |
|------|---------|-------------|------|
| Zookeeper | 2181 | **2201** | 协调服务 |
| Kafka | 9092 | **9112** | 消息队列 |
| Redis | 6379 | **6399** | 缓存/会话 |
| PostgreSQL | 5432 | **5452** | 主数据库 |
| Kafka UI | 8080 | **8100** | 消息队列管理界面 |
| API Service | 8000 | **8020** | 后端 API |

## Python 依赖管理规范

### 核心原则
1. **完全隔离** - 每个项目必须使用独立的虚拟环境或 Docker
2. **禁止全局安装** - 不允许使用 `pip install` 全局安装包
3. **版本锁定** - 必须锁定依赖版本，禁止自动更新
4. **审计跟踪** - 所有依赖变更需记录原因

### 实施要求
- 使用 `requirements.txt` 锁定版本
- 使用 `requirements-dev.txt` 区分开发依赖
- 使用 Docker 时提供明确的镜像版本标签
- 依赖变更需记录在 `docs/dependency-changes.md`
